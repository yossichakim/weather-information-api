## Plan Requirements

For broad repository changes, create a dedicated execution plan under `docs/plans/`.

Each plan should include:

- current state
- desired outcome
- affected files
- architecture or implementation approach
- milestones
- validation commands
- risks
- decisions
- progress checklist

Plans must remain factual, reflect the actual repository state, and be updated throughout implementation.
